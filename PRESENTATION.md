# Сценарий презентации — АВПК Общежитие

## Подготовка (до презентации)

```bash
# Терминал 1 — бэкенд
cd ~/ahpc-dormitory/backend && node index.js

# Терминал 2 — CMS (админ-панель)
cd ~/ahpc-dormitory/cms && npx vite --port 5173

# Терминал 3 — фронтенд (для студентов)
cd ~/ahpc-dormitory/frontend && npx vite --port 5174
```

- CMS: http://localhost:5173 → логин `admin` / `admin123`
- Фронтенд: http://localhost:5174

## Возможные вопросы комиссии и ответы

**Q: Как защита от дубликатов?**
**A:** Два уровня — проверка занятости места + fuzzy-поиск по телефону и имени (алгоритм Левенштейна, ≥80% сходства). Подозрительные заявки не блокируются, а помечаются флагом для администратора.

**Q: Что если студент выселится и снова заселится?**
**A:** При выселении проставляется `movedOut`. Он может подать новую заявку или быть заселённым администратором в другую комнату — старая запись остаётся в истории.

**Q: Как работает план этажа?**
**A:** Комнаты отрисованы в SVG, координаты привязаны к реальному плану здания через Figma. При наведении — подсветка и всплывающая информация о занятости.

**Q: Сложно ли добавить новую специальность или группу?**
**A:** Нет — справочники не зашиты в код. Достаточно завести студента с новой группой в CMS, система подхватит автоматически.

**Q: Можно ли выгрузить отчёт за прошлый месяц?**
**A:** Да, в модуле оплаты можно выбрать любой месяц и год, после чего выгрузить в Excel с форматированием и автофильтром.
